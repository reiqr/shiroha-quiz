# 解析器回归测试说明

## 是什么

`test/native-parser-regression/` 是原生解析器的外部回归核对包。不修改 App 源码、不依赖现有单元测试。它直接用真实题库样例跑一遍当前解析器，将结果和预期对比，检查有没有误伤。

当前回归包已覆盖 49 个样例，重点保护“标准格式优先、复杂格式兜底”的原则：标准题库不能被材料题、答案区、代码表达式、主观题续写等复杂规则误伤；复杂题库则通过专门样例验证兜底能力是否仍然可用。

## 怎么运行

常用入口有两个：

- 在仓库根目录执行 `run-regression.ps1`，适合从项目根目录一键跑。
- 在 `test/native-parser-regression/` 目录执行 `run-external-regression.ps1`，适合只维护外部回归包时使用。

仓库根目录：

```powershell
.\run-regression.ps1
```

外部回归目录：

```powershell
cd test\native-parser-regression
.\run-external-regression.ps1
```

如果需要拆开排查，也可以直接运行 Gradle + 对比脚本：

```bash
bash apps/android/gradlew -p test/native-parser-regression/runner run
python test/native-parser-regression/tools/compare_regression.py
```

## 怎么加新样例

1. 在 `test/native-parser-regression/samples/` 放一份 `.txt`（`tsv` 也可）作为导入原文
2. 先在 `expected/` 放一份空的同名 `.json`，跑一次看实际输出
3. 核对实际输出没问题后，把 `actual/<id>.json` 复制到 `expected/`
4. 在 `manifest.json` 补一条用例记录，标注 `risk`（防什么类型的误伤）

## 怎么看结果

- 控制台直接显示 `passed` / `failed` 计数
- 失败用例会在 `actual/REGRESSION_REPORT.md` 里附实际题目摘录，方便定位差异
- 全部通过时脚本正常退出；任何失败都会返回非 0 退出码

## 什么时候跑

- 修改了解析器（parser/score/validate 下任何文件）
- 新增了题型识别逻辑
- 修改了答案区、双文件、材料题、主观题续写或 AI 导入预览相关逻辑
- 发版前最后一轮校验

## 重点覆盖范围

- 标准单选、多选、判断、填空、简答题。
- 双文件答案匹配、范围答案、紧凑答案行和行内多条答案。
- 多空填空、内联空位、分区多空继承。
- 代码表达式、Shell 文本、英文缩写和医学缩写，避免被误判为选项或答案。
- 集中答案解析区，以及答案区后继续出现正文题目的恢复。
- 解析正文中的编号分步，避免 `1. 先... / 2. 再...` 被切成新题。
- 共用题干 / 材料题导语，避免材料说明被误判为独立题目。
