# 解析器回归测试剩余关注项

> 更新日期：2026-07-07
> 当前状态：`manifest.json` 维护 63 个有效 parser 用例，编号已扩展到 64；其中历史编号 35 暂未启用。

这份文件只记录当前仍需关注或暂缓的点。已经进入稳定回归的内容，以 `test/native-parser-regression/manifest.json` 为准。

## 已覆盖重点

- 标准单选、多选、判断、填空、简答题。
- 双文件答案合并，包括题号匹配、章节分区、乱序答案和答案表。
- 答案表、范围答案、紧凑答案、行内多答案。
- 判断题常见答案写法，例如 `正确 / 错误 / 对 / 错 / √ / × / ✓ / ✗`。
- 答案解析区跳过、误入答案区后恢复、真正答案区不误恢复。
- 解析正文中的分步编号保护，避免把 `1. 先... 2. 再...` 切成新题。
- 主观题答案中的 `10.` 这类分项保护，避免误切题。
- 医学缩写、英文缩写、分区标题误判等真实边界。

## 已收口问题

- 区间答案数量不匹配时，不再静默只填前几题。
- `upsertEntry` 空类别不再跨类别模糊覆盖。
- 解析续行中的 `A项正确 / B项错误` 不再误作答案条目。
- `一百零三` 等中文题号可正确解析。
- 双文件存在明确题号交集时，优先题号合并，避免顺序兜底串答案。
- 答案区没有前置题号上下文时，不再轻易恢复成正文题目。
- 答案区兜底路径避免生成 `1. 1. A` 这类重复编号文本。

## 剩余暂缓项

| 优先级 | 项目 | 当前判断 |
| --- | --- | --- |
| 中 | 大文档 `prefixBlocks` 二次复杂度 | 性能优化项，真实大卷出现卡顿时再处理。 |
| 低 | `A 内容` 这种空格分隔选项格式 | 容易误伤英文句子，建议先补真实样例再改。 |
| 低 | 无编号且以 `。` 结尾的行判断过宽 | 涉及无编号题支持，不宜在没有样例时扩大规则。 |
| 低 | `cleanup()` 修剪链条复杂 | 重构项，不影响当前导入结果。 |
| 低 | 多选答案只剩一个选项时自动降级单选 | 产品策略项，暂不建议自动改题型。 |

## 后续维护原则

1. 每个新增支持尽量同时补一个正向样例和一个反向样例。
2. 标准题库解析永远优先，复杂兜底不能污染正常标准格式。
3. 性能优化、结构重构和行为修复分开做，避免一次改动引入难以定位的回归。
